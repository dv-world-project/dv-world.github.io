Abstract

Benchmark Scope And Motivation

DV-World is designed to close the gap between lightweight chart-generation benchmarks and the actual lifecycle of professional visualization work.

Abstract Summary

Existing benchmarks often rely on code sandbox confinement, single-language creation tasks, or perfect-intent assumptions. DV-World instead evaluates data visualization agents in realistic software-grounded settings that require spreadsheet manipulation, cross-modal evolution, and proactive clarification.

  • DV-Sheet covers native spreadsheet chart creation, diagnostic repair, and dashboard composition.
  • DV-Evolution evaluates adaptation from reference visuals and datasets into executable outputs across diverse frameworks.
  • DV-Interact evaluates whether agents ask clarifying questions and align to ambiguous user intent.
  • The paper reports that state-of-the-art models still achieve less than 50% overall performance.

Benchmark Construction

Three domains in the paper

The benchmark definition in the paper spans native spreadsheet manipulation, cross-modal logic evolution, and interactive clarification under ambiguity.

DV-Sheet

Native spreadsheet manipulation

The agent edits spreadsheets directly to create charts, repair defective visualizations, and compose dashboards under native workbook constraints.

  • Direct range binding and object-level editing.
  • Repair tasks judged as precise interventions, not regenerated screenshots.
  • Dashboard tasks emphasize insightfulness, not just rendering.
DV-Evolution

Cross-modal logic evolution

The agent must reverse-engineer visual semantics from reference artifacts and evolve them into executable outputs for new data and new languages.

  • Library-aware transfer instead of raw code generation.
  • Highlights portability gaps and layout mismatches.
  • Perfect fit for wrong-vs-better visual storytelling.
DV-Interact

Proactive clarification under ambiguity

The agent interacts with a user-simulator based environment and must resolve ambiguity through legitimate clarification rather than assumption-first execution.

  • User-simulator driven evaluation.
  • Measures inquiry deficits and alignment behavior.
  • Exposes interactive avoidance clearly and visually.
